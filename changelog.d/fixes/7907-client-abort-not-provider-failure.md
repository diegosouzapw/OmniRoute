- **Resilience**: client-initiated aborts no longer count as provider failures (#7907) — `abort(reason)` rejects the upstream fetch with a raw string reason (e.g. `request_signal_aborted`) that carried no `name`/`status`, so it defaulted to HTTP 502 and cooled down the healthy connection while accruing whole-provider circuit-breaker failures; a new `isClientAbortError` classifier normalizes every client-abort shape to 499 in the chatCore failure path and is excluded from the provider breaker's `isFailure` predicate
