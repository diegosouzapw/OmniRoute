# Friction Log

> Track developer experience friction points so they can be addressed systematically.

## How to Use

When you encounter unnecessary friction during development, add an entry here:

```markdown
### YYYY-MM-DD — [Brief Title]

**Area**: [e.g., build, test, deploy, CI, docs, dev loop]
**Severity**: [minor | annoying | blocking]
**Time Lost**: [e.g., 15min, 2h]

**What happened:**
Describe the friction — what did you try to do, and what got in the way?

**Suggested fix:**
What change would prevent this friction for future developers?

**Fixed in**: [PR # or commit, if addressed]
```
