# side-43: Structured logging on API responses

**Date:** 2026-06-20
**Scope:** API response / request logging in active backend services (Rust `tracing`, Go `slog`/`log`, Python `structlog`/`logging`).

## What I checked

```bash
# 1. Go services — structured slog vs unstructured log.Printf
grep -nE "tracing|slog|Info|Warn" ./KWatch/server/handlers.go
# → no slog; only fmt.Fprintf / map[string]string assembly; no slog

grep -n "log\." ./KWatch/server/server.go
# Lines 48-71: log.Printf("Starting kwatch HTTP server on %s:%d", ...) etc.
# Unstructured key=value (printf format strings), not slog.

# 2. AuthKit — the only structured-logging exemplar
grep -nE "slog|tracing|log\." ./AuthKit/go/middleware/middleware.go
# Lines 437-444: logger.Info("http_request", slog.String("method", r.Method), ...)
# Structured slog with key/value pairs: method, path, status, duration,
#   correlation_id, client. EXEMPLARY.

# 3. BytePort — unstructured log.Printf in API response path
grep -nE "log\." ./BytePort/backend/byteport/lib/apilink.go
# Line: log.Printf("Portfolio API Response: %s\n", string(body))
# Logs the raw response body to stdout — UNSAFE (PII, secrets risk).

grep -nE "log\." ./BytePort/backend/byteport/lib/git.go
# log.Printf("Response Token: [REDACTED]\n")  — at least redacts tokens
# log.Printf("Response Refresh Token: [REDACTED]\n")
# log.Printf("Response Expiry: %s\n", response.TokenExpiry)
```

## Findings

**Compliant (structured, key/value):**
- `AuthKit/go/middleware/middleware.go:438-444` — `logger.Info("http_request", slog.String("method", r.Method), slog.String("path", r.URL.Path), slog.Int("status", status), slog.Duration("duration", duration), slog.String("correlation_id", GetCorrelationID(r.Context())), slog.String("client", r.RemoteAddr))`. This is the gold standard. Slog key/value, includes correlation_id, safe (no body logged).
- `phenotype-python-sdk/packages/observability-kit/rust/phenotype-health-runtime/src/http.rs` — axum `health_handler` uses `tracing` crate. Compliant.

**Non-compliant (printf-style, no fields):**
- `KWatch/server/handlers.go` — no slog/tracing in handler functions; HTTP responses are assembled with `json.NewEncoder(w).Encode(...)` and `map[string]string` but emit no structured log line per request. The server startup log is `log.Printf("Starting kwatch HTTP server on %s:%d", ...)` (line 48 of `server.go`) — unstructured. The `/health` and `/status` endpoints (handlers.go:140-142, "GET %s/quick - Ultra-fast health check") log nothing.
- `KWatch/server/server.go:48-71` — `log.Printf` for startup/shutdown; no per-request structured log.
- `BytePort/backend/byteport/lib/apilink.go` — `log.Printf("Portfolio API Response: %s\n", string(body))` logs the full HTTP response body to stdout. This is **doubly bad**: (a) unstructured, (b) leaks PII/secrets/tokens depending on API contract.

**Mixed (some structured, some not):**
- `KWatch/config/config.go:275` — uses a custom `Config` struct; no slog but also no request log. Acceptable for config but missing for HTTP path.

## Severity

**P1.** The lack of structured request logs in `KWatch` and `BytePort` means request correlation, error tracking, and SLO analysis cannot be done without parser-specific regex on the printf format. Industry standard (CNCF Logging SIG, OpenTelemetry logs spec) requires key/value. BytePort's `lib/apilink.go` is a **P0** for the response-body logging: any token, password, or PII returned by the Portfolio API is now in stdout logs, which is a leak. AuthKit and the observability-kit crates are exemplary.

## Recommended action

1. **P0 immediate:** `BytePort/backend/byteport/lib/apilink.go` — replace `log.Printf("Portfolio API Response: %s\n", string(body))` with `log.Printf("Portfolio API call complete: status=%d bytes=%d", resp.StatusCode, len(body))` (no body). If body logging is required, redact via `slog.String("body_redacted", "[truncated]")` and document in code why the body is required.
2. **P1 this sprint:** `KWatch/server/handlers.go` and `KWatch/server/server.go` — add a request-logging middleware that emits `slog.Info("http_request", method, path, status, duration_ms, correlation_id)` (mirroring AuthKit's pattern). Reuse AuthKit's `observabilityMiddleware` if extractable to a shared module.
3. **P1 this sprint:** migrate `KWatch` to `log/slog` (Go 1.21+ stdlib) and replace every `log.Printf` with `slog.Info`/`slog.Error` carrying structured fields.
4. **Policy gate:** add `policy-gate.yml` rule that flags any HTTP handler without an associated request-logging middleware. (ADR-040 gate.)
5. **Document:** add `docs/logging-policy.md` requiring structured logs (slog / tracing / structlog) for any HTTP request/response path; printf-style is permitted only at startup/shutdown.
