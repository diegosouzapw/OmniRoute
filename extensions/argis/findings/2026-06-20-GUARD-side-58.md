# Guardrail: CI check for known CVEs in dependencies (side-58)

**Date:** 2026-06-20
**Owner:** pheno-ci-templates circle
**Status:** Proposed (FLEET_DAG.db: side-58 DONE — file artifact pending)

## Current state

`cargo audit` is run monthly per ADR-042 but not on every PR. A new
advisory published Tuesday can land in `main` Wednesday and ship Thursday
before the monthly sweep catches it. The RustSec advisory DB updates
within hours of CVE disclosure; we are leaving free lead-time on the
table.

## Proposed guardrail

Run `cargo audit` on every PR and on push to `main`, gating merge on a
clean result. The check also runs `pip-audit` for Python crates
(`pyproject.toml` workspace members) and `govulncheck` for Go modules
where they exist. Failure modes: `deny warnings` for `main`, `deny
unacknowledged` for PRs.

## Concrete implementation

`.github/workflows/audit.yml` (14 LOC):

```yaml
name: deps-audit
on:
  pull_request:
  push: { branches: [main] }
jobs:
  audit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - run: cargo install --locked cargo-audit
      - run: cargo audit --deny warnings
      - uses: actions/setup-python@v5
      - run: pip install pip-audit && pip-audit --strict
      - uses: actions/setup-go@v5
        with: { go-version: stable }
      - run: |
          go install golang.org/x/vuln/cmd/govulncheck@latest
          govulncheck ./...
```

## Effort

**S** — 15 minutes per repo. Combined with side-06 (deny.toml) and the
monthly ADR-042 sweep, this gives three layers: PR-gate (this), CI-weekly
(side-06), monthly-cadence (ADR-042).
