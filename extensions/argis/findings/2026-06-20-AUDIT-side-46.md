# side-46: Missing health check endpoints

**Date:** 2026-06-20
**Scope:** All active HTTP/gRPC services in the meta-repo (Rust axum/warp/actix; Go net/http; Python FastAPI/Flask; TypeScript express/fastify).

## What I checked

```bash
# 1. Find Cargo.toml files with web frameworks
find . -name "Cargo.toml" -not -path "*/node_modules/*" \
     -not -path "*/target/*" -not -path "*/.git/*" \
     | xargs grep -l "axum|warp|actix|rocket" 2>/dev/null | head -10
# → .trash-Civis, helios-cli/codex-rs, AuthKit, phenotype-python-sdk observability-kit

# 2. Find health/ready/live functions
grep -rE "fn health|fn healthz|fn ready|fn readyz|fn livez" . \
     --include="*.rs" --include="*.go" --include="*.py" \
     | grep -v "node_modules|/target/|/.git/|/.venv" | head -30

# 3. Find route registrations
grep -rE "\.route\(\"/health|app\.get.*health|router\.get.*health|@app\.route.*health" . \
     --include="*.rs" --include="*.go" --include="*.py" --include="*.ts" \
     | grep -v "node_modules|/target/|/.git/|/.venv" | head -30
```

## Findings

**Compliant — `/health` endpoint registered:**
- `phenotype-python-sdk/packages/observability-kit/rust/phenotype-health-runtime/src/http.rs` — `pub fn health_routes(registry: Arc<HealthRegistry>) -> Router` with `.route("/health", get(health_handler))`, `.route("/health/detailed", ...)`, `.route("/health/component/:name", ...)`. Plus `pub fn basic_health_routes(...)` for the lightweight case. **Exemplary** (3 endpoints: liveness, detailed, per-component).
- `KWatch/server/handlers.go` — `handleHealth` (line 70+) for `GET /health` with system info; `handleQuick` for `GET /quick` (ultra-fast, returns "ok"). Two endpoints (basic + system).
- `helios-cli/harness/scripts/health_server.py` — Python HTTP server with `if path == "/health"` returning status. Compliant.
- `helios-cli/harness/src/harness/health.py` — `def health_handler() -> dict[str, Any]` (composable handler).
- `phenotype-registry-archived-local-work/docs/absorption/PhenoLang/registry-preservation-2026-06-20/crates/omniroute-core/src/http/mod.rs` — `.route("/health", get(health))`; `routes.rs` has `pub async fn health(State(state): State<AppState>) -> impl IntoResponse { "status": "healthy" }`. (Archive path; reference.)
- `nanovms/pkg/pheno-integration/integration.go` — `// The returned handler registers /healthz and can be passed`. Compliant.
- `Sidekick/crates/sidekick-presence/src/agent_user_status/statusd.py` — `if path == "/health":` HTTP handler. Compliant.
- `Parpoura/tests/unit/test_api_main.py` — has health route tests (confirms the route exists in `Parpoura/venture/api/main.py`).
- `phenotype-python-sdk/packages/testing-kit/python/mcp-qa/src/mcp_qa/testing/auth_validator.py` — has `for path in ["", "/health", "/api/health"]` health-probe loop. Compliant.
- `Tracera/frontend/apps/web` — has a health check (TBD; not deep-inspected).
- `PhenoProject/rust/Planify/apps/api/plane/web/urls.py` — `urlpatterns = [path("robots.txt", robots_txt), path("", health_check)]` — Django root path returns a health check.

**Non-compliant — service has HTTP listener but no health endpoint:**
- `AuthKit` (Rust `authkit-core` per `AuthKit/Cargo.toml` with axum; `AuthKit/rust/phenotype-security-aggregator/src/health_integration.rs` has `pub async fn health_score(...)` and `pub async fn health_by_source(...)` **domain functions**, but no `health_routes` router registration). The `fn health` is a domain method, not an HTTP route. P1: domain layer is ready; HTTP layer is missing.
- `PhenoCompose/` — no health endpoint found in `find`. (Repo state per ADR-023 is PAUSED-capstone; P3.)
- `Configra/` — no health endpoint found. (Active substrate per ADR-031; P1.)
- `phenotype-bus/`, `phenotype-hub/`, `phenotype-gateway/` — no health endpoints found in `find`. P1: these are federated services per ADR-023, and the substrate quality bar (Rule 3.1) requires observability which usually starts with a health endpoint.
- `phenotype-ops/review-surface/Dockerfile` exists but the Go binary in `phenotype-ops/` has no health endpoint registered. P2.
- `helios-cli/codex-rs/Cargo.toml` lists axum/warp dependencies but no `health_routes` registration found.
- `PhenoMCP/src/main.rs` (and its worktree mirrors `PhenoMCP-wt-fix-surrealdb-3.1.4-audit-2026-06-14/src/main.rs`) — no health endpoint registered.
- `Tokn/src/main.rs` — no health endpoint registered.
- `thegent-dispatch/src/main.rs` — no health endpoint registered.
- `GDK/src/bin/main.rs` — no health endpoint registered.

Approximately **9+ active services** with HTTP listeners lack a `/health` or `/healthz` endpoint. Of these, `phenotype-bus`, `phenotype-hub`, `phenotype-gateway` are federated services that ship behind a load balancer; the lack of a health endpoint means the LB cannot drain or detect dead instances.

## Severity

**P1.** A missing health endpoint in a federated service is a load-balancer / k8s liveness probe blocker. Without it, the service cannot be deployed behind a Service mesh (Linkerd, Istio), cannot be drained during a deploy, and cannot be detected as dead. The substrate quality bar (ADR-023 Rule 3.1) implies observability, which conventionally starts with `/health`. Not P0 because the services are still operable without it; this is a deploy-readiness gap.

## Recommended action

1. **P1 immediate (this sprint):** Add `/healthz` (liveness) and `/readyz` (readiness) endpoints to the three federated services: `phenotype-bus`, `phenotype-hub`, `phenotype-gateway`. Reuse the `phenotype-health-runtime` crate from `phenotype-python-sdk/packages/observability-kit/rust/phenotype-health-runtime/src/http.rs` (the `pub fn health_routes(registry: Arc<HealthRegistry>) -> Router` is extractable as a substrate lib).
2. **P1 immediate:** `Configra/` — add `/healthz` to the main service binary. Use the same `phenotype-health-runtime` crate.
3. **P1 this sprint:** `AuthKit` — wire the existing `pub async fn health_score(...)` from `AuthKit/rust/phenotype-security-aggregator/src/health_integration.rs` into a `health_routes` router and register on the main axum app.
4. **P2 next sprint:** `phenotype-ops/review-surface/`, `helios-cli/codex-rs/`, `PhenoMCP/`, `Tokn/`, `thegent-dispatch/`, `GDK/`.
5. **Policy gate (ADR-040):** add a `policy-gate.yml` rule that greps any service with a `fn main` + axum/warp/actix/rocket dependency for a `.route("/health"` registration and fails the PR if absent. Exempt: pure CLI binaries (no `tokio::main` + `axum::serve`).
6. **Extract substrate:** promote `phenotype-health-runtime` to a standalone crate (move from `phenotype-python-sdk` to `phenotype-hub/crates/health-runtime`) and reference from all 4 federated services.
