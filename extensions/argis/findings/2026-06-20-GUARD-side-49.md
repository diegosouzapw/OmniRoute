# Guardrail: commit-msg hook for Conventional Commits (side-49)

**Date:** 2026-06-20
**Owner:** pheno-ci-templates circle
**Status:** Proposed (FLEET_DAG.db: side-49 DONE — file artifact pending)

## Current state

`AGENTS.md` ("Conventions") specifies Conventional Commits
(`feat:`, `fix:`, `chore:`, etc.) but there is no local hook enforcing it.
Reviewers catch the occasional `wip` or `updated stuff` commit at PR time,
which is too late — `git log --oneline` becomes a soup and `cliff` (used
for changelog generation) emits garbage.

## Proposed guardrail

A `commit-msg` hook via the existing `lefthook.yml` (already in fleet use
per `lefthook.yml` at the repo root) wired to `commitlint` with the
`@commitlint/config-conventional` preset. Local rejection, friendly error,
no push, no CI waste.

## Concrete implementation

`lefthook.yml` snippet (5 LOC, append to existing):

```yaml
commit-msg:
  commands:
    commitlint:
      run: npx --no-install commitlint --edit "$1"
      glob: "*.{ts,js,json}"
```

`commitlint.config.js` (5 LOC):

```js
module.exports = {
  extends: ['@commitlint/config-conventional'],
  rules: { 'header-max-length': [2, 'always', 100] },
};
```

## Effort

**S** — 10 minutes per repo. lefthook is already installed; add the two
config files and run `lefthook install` once.
