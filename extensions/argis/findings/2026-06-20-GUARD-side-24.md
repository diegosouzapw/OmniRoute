# Guardrail: input validation fuzzing for connector manifests (side-24)

**Date:** 2026-06-20
**Owner:** pheno-connectors circle
**Status:** Proposed (FLEET_DAG.db: side-24 DONE — file artifact pending)

## Current state

Connector manifests (`ConnectorManifest` YAML files) are deserialized via
`serde_yaml` then passed to `validate()`. The validator has hand-rolled
checks for "obvious" bad input — empty name, negative timeout — but
adversarial YAML (alias bombs, billion-laughs, recursive anchors, deeply
nested mappings) is not exercised. A 2026-04 incident showed a single 2KB
manifest could trigger >2s parse time.

## Proposed guardrail

A `cargo-fuzz` target that generates arbitrary YAML bytes and times
`validate()` with a 100ms hard ceiling. Build a regression corpus from the
2026-04 incident payload + a curated set of pathological inputs
(`yaml-anchor-loop`, `billion-laughs`, `unicode-bidi-attack`).

## Concrete implementation

`fuzz/fuzz_targets/manifest.rs` (10 LOC):

```rust
#![no_main]
use libfuzzer_sys::fuzz_target;
use pheno_connectors::{ConnectorManifest, validate};
use std::time::{Duration, Instant};

fuzz_target!(|data: &[u8]| {
    let t = Instant::now();
    if let Ok(m) = serde_yaml::from_slice::<ConnectorManifest>(data) {
        let _ = validate(&m);
    }
    assert!(t.elapsed() < Duration::from_millis(100),
        "manifest validation exceeded 100ms budget");
});
```

`Cargo.toml` dev-deps (3 LOC):

```toml
[dev-dependencies]
libfuzzer-sys = "0.4"
```

## Effort

**M** — half-day for the target, 1h to seed the corpus, runs nightly in CI
at <2min/repo.
