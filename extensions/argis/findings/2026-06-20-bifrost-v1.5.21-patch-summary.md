# Bifrost v1.5.21 Patch Summary

**Date:** 2026-06-20
**Purpose:** T1.4 — patch summary for 9 bifrost server plugins.
**Refs:** `findings/2026-06-20-bifrost-v1.5.21-server-enumeration.md`, ADR-050 §8 ACCEPTED

## Summary

Bifrost SDK bump v1.2.30 → v1.5.21 affects **6 of 9** server plugins. Three SDK breaks are present; two servers (`cache`, `budget`, `observability`) are unaffected because they don't use the affected Bifrost APIs.

## Affected servers (6)

| Server | SDK breaks | Patch file | Estimated LoC change |
|--------|------------|------------|----------------------|
| `smartfallback` | MissingBatchPayloadKey, BifrostExecutor | `2026-06-20-bifrost-v1.5.21-patch-smartfallback.patch` | ~30 |
| `contentsafety` | BifrostExecutor | (v12 sub-task) | ~20 |
| `ratelimit` | stream-compat | (v12 sub-task) | ~25 |
| `audit` | MissingBatchPayloadKey | (v12 sub-task) | ~20 |
| `tracing` | BifrostExecutor | (v12 sub-task) | ~25 |
| `retry` | BifrostExecutor | (v12 sub-task) | ~20 |
| **Total** | | | **~140 LoC** |

## Not affected (3)

| Server | Reason |
|--------|--------|
| `cache` | Uses Bifrost only as cache-key hasher; no Execute / BatchPayload / StreamChunk |
| `budget` | Cost-only enforcement; no Bifrost execute path |
| `observability` | Read-only metrics scrape; no Bifrost SDK call |

## Regression test plan

1. **Unit tests** — existing per-server tests run with `cargo test -p bifrost-<server>`.
2. **Integration tests** — `cargo test -p bifrost-integration` exercises the full plugin chain.
3. **Compatibility shim** — patch each server with the v1.5.21 SDK calls; add `// SDK: bifrost ^1.5.21` marker to Cargo.toml.
4. **Migration PR** — single PR `KooshaPari/Bifrost:feat/bifrost-sdk-1.5.21-bump-2026-06-20` bundles all 6 patches.
5. **Smoke test** — start Bifrost gateway; verify each plugin responds on the canonical endpoint `/v1/<plugin>/health`.

## Rollout order

1. Land the 3 "no-API-change" servers as-is (cache, budget, observability) — zero-risk.
2. Land `audit` patch first (lowest blast radius — read-only side effect).
3. Land `tracing`, `retry`, `contentsafety` (medium blast radius).
4. Land `smartfallback` and `ratelimit` last (highest blast radius — routing-critical).
