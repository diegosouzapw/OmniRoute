# Guardrail: CI check for unused dependencies (cargo-udeps) (side-57)

**Date:** 2026-06-20
**Owner:** pheno-ci-templates circle
**Status:** Proposed (FLEET_DAG.db: side-57 DONE — file artifact pending)

## Current state

Crates accumulate dev-dependencies and `[features]`-gated deps that are no
longer referenced. Removing them is tedious and easy to defer, so they
ship. `pheno-agents-md` carries `syn = "2.0"` from a removed proc-macro
experiment that nobody noticed for 4 months. Build time, attack surface,
and licensing review scope all grow proportionally.

## Proposed guardrail

A CI step that runs `cargo udeps --workspace --all-targets` on every PR.
The tool walks the dep graph via `cargo metadata` and `RUSTFLAGS`, flags
anything declared but unreferenced. Requires `nightly` (udeps uses
`-Z build-std`-adjacent internals); pin the toolchain version in the
workflow.

## Concrete implementation

`.github/workflows/udeps.yml` (10 LOC):

```yaml
name: udeps
on: [pull_request]
jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@nightly
      - run: cargo install --locked cargo-udeps
      - run: cargo udeps --workspace --all-targets
        env:
          RUSTFLAGS: --cfg=nightly
```

## Effort

**S** — 15 minutes per repo. False positives on `[features]`-only deps are
handled by adding `package.metadata.cargo-udeps.ignore = ["dep-name"]` to
the offending `Cargo.toml` — well-documented in the tool's README.
