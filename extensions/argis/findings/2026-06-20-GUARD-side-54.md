# Guardrail: enforce README freshness check on release (side-54)

**Date:** 2026-06-20
**Owner:** pheno-ci-templates circle
**Status:** Proposed (FLEET_DAG.db: side-54 DONE — file artifact pending)

## Current state

A `Cargo.toml` version bump can ship without the `README.md` quickstart
being updated to match the new version. The published docs link to
outdated install snippets (`pheno-cli = "0.4"` when the crate is at `0.7`).
Caught only when a downstream user opens an issue. Of the 23 crates
audited in `findings/2026-06-19-L5-110-112-*-audit.md`, 9 had stale
README quickstart versions.

## Proposed guardrail

A CI check that runs on `tag` (release) and fails the publish step if
`README.md` was not modified more recently than the most recent
`Cargo.toml` version bump. Catches the case where version changed but
README didn't.

## Concrete implementation

`.github/workflows/release.yml` freshness step (8 LOC):

```yaml
- name: README freshness
  if: startsWith(github.ref, 'refs/tags/v')
  run: |
    README_MTIME=$(git log -1 --format=%ct README.md)
    CARGO_MTIME=$(git log -1 --format=%ct Cargo.toml)
    if [ "$README_MTIME" -lt "$CARGO_MTIME" ]; then
      echo "::error::README.md (mtime $README_MTIME) is older than \
            Cargo.toml ($CARGO_MTIME) — update quickstart before release"
      exit 1
    fi
```

**S** — drop in the step. Whitelist via CODEOWNERS-style override comment
