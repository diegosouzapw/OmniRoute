# Guardrail: CI check for TODO comment resolution (side-55)

**Date:** 2026-06-20
**Owner:** pheno-ci-templates circle
**Status:** Proposed (FLEET_DAG.db: side-55 DONE — file artifact pending)

## Current state

PRs add `TODO`, `FIXME`, and `XXX` comments as breadcrumbs and forget them.
A 2026-05 sweep of the fleet found **412 open TODOs** across 17 crates; 180
of them were over a year old and had no tracking issue. The convention
"every TODO needs a tracking issue" exists in `AGENTS.md` but is not
enforced.

## Proposed guardrail

A CI step that diffs the PR against the base branch and fails if any **new**
`TODO`/`FIXME`/`XXX` line is added without an inline reference to an issue
or ADR (`TODO(#1234)` or `TODO(ADR-040)`). Tracking format is enforced by
regex. Existing TODOs are not flagged; only new additions.

## Concrete implementation

`.github/workflows/todo-check.yml` (10 LOC):

```yaml
name: todo-check
on: [pull_request]
jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0 }
      - run: |
          UNTRACKED=$(git diff origin/${{ github.base_ref }}...HEAD \
            | grep -E '^\+.*(TODO|FIXME|XXX)\b' \
            | grep -vE 'TODO\(#?[0-9]+\)|TODO\(ADR-[0-9]+\)')
          if [ -n "$UNTRACKED" ]; then
            echo "::error::Untracked TODO/FIXME/XXX added:"
            echo "$UNTRACKED"
            exit 1
          fi
```

## Effort

**S** — 20 minutes to roll out. Initial sweep: run once to enumerate the
existing 412 TODOs as the baseline; only delta from baseline is checked
after that.
