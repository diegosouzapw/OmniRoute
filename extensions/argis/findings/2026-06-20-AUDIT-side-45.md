# side-45: Reversible DB migrations

**Date:** 2026-06-20
**Scope:** All `*.sql` files in non-vendored, non-trash paths. Distinguishes single-file migrations (with embedded `-- DOWN` markers) from up/down pair migrations.

## What I checked

```bash
# 1. Find all SQL files
find . -name "*.sql" -not -path "*/.git/*" -not -path "*/.venv/*" \
     -not -path "*/node_modules/*" | head -100
# → 100+ migration/schema files across the meta-repo

# 2. Find migrations that have -- DOWN markers (reversible single-file)
grep -rE "DOWN|down\.sql|-- DOWN" . --include="*.sql" \
     | grep -v "node_modules|/.git/" | head -20
# → pheno/agileplus-sqlite/src/migrations/* and pheno/agileplus/.../migrations/*
#   each have -- DOWN markers (good pattern)

# 3. Find SQL files that LACK any -- DOWN marker
find . -name "*.sql" -not -path "*/.git/*" -not -path "*/.venv/*" \
     -not -path "*/node_modules/*" | xargs grep -L "DOWN|-- DOWN"
# → ~20+ non-reversible SQL files (see findings)

# 4. Find up/down paired migrations
find . -name "*down*.sql" -not -path "*/.git/*"
# → forgecode/crates/forge_repo/src/database/migrations/*/down.sql
# → db/migrations/00000[1-5]_*.down.sql
# → argis-extensions/db/migrations/00000[1-5]_*.down.sql
```

## Findings

**Compliant — paired up/down (best practice, expand-contract compatible):**
- `forgecode/crates/forge_repo/src/database/migrations/2025-09-12-065405_create_conversations_table/down.sql` (and 12 more `down.sql` files in the same dir; each has a sibling `up.sql`).
- `db/migrations/000001_initial_schema.down.sql`, `000002_provider_accounts.down.sql`, `000003_documents.down.sql`, `000004_fuzzy_search.down.sql`, `000005_advanced_extensions.down.sql` (5 reversals).
- `argis-extensions/db/migrations/00000[1-5]_*.down.sql` (5 reversals, mirrors `db/migrations`).
- `phenotype-registry-archived-local-work/db/migrations/...` (mirror, also reversible).

**Compliant — single-file with `-- DOWN` markers (acceptable, less preferred):**
- `pheno/crates/agileplus-sqlite/src/migrations/001_create_features.sql` — has `-- DOWN` block.
- `pheno/crates/agileplus-sqlite/src/migrations/002_create_work_packages.sql` — has `-- DOWN`.
- `pheno/crates/agileplus-sqlite/src/migrations/003_create_governance_contracts.sql` — has `-- DOWN`.
- `pheno/crates/agileplus-sqlite/src/migrations/004_create_audit_log.sql` through `017_create_projects.sql` (all 17 single-file reversible).
- `pheno/agileplus/crates/agileplus-sqlite/src/migrations/...` (same 17, mirrored). Good.

**Non-compliant — irreversible (no `DOWN`, no `down.sql`):**
- `helios-cli/migrations/0001_init.sql` — single forward-only file, no `0001_init.down.sql` companion. P1 because this is the production schema; rollback requires manual SQL reconstruction.
- `Tokn/ledger/unified_model_provider_pareto.view.sql` — view only; technically reversible via `DROP VIEW`, but no `-- DOWN` block.
- `Tokn/ledger/unified_model_provider_ledger.schema.sql` — schema DDL, no `DOWN`.
- `Tokn/ledger/unified_model_provider_ledger.seed.sql` — seed data, no `DOWN` (acceptable: seeds are forward-only by design; but should be marked `-- NO-DOWN: seed data`).
- `Tokn/models_schema_seed.sql` — seed, no `DOWN`.
- `pheno/crates/agileplus-sqlite/src/migrations/016_git_bindings.sql` — NO `DOWN` marker (the only one of the 17 in this dir that is irreversible; appears to be a late addition that bypassed the convention).
- `pheno/agileplus/crates/agileplus-sqlite/src/migrations/016_git_bindings.sql` — same as above (mirror).
- `KaskMan/architecture/database-schemas.sql` — design doc, not a migration; exempt.
- `FocalPoint/AtomsBot-2nd/db/schema.sql` — single forward-only.
- `FocalPoint/kwality-wtrees/gitignore-pycache/database/schema/001_initial_schema.sql` — single forward-only.
- `.trash-Civis-*` (multiple) — `crates/infra/migrations/0001_create_replays.sql` no DOWN. Trashed; out of scope.

Approximately **10+ active SQL files lack reversibility**, including the production `helios-cli/migrations/0001_init.sql` (no rollback path). `pheno/.../016_git_bindings.sql` is the only file in an otherwise-compliant migration set that breaks the convention.

## Severity

**P1.** Irreversible migrations are a well-known production risk (cf. GitLab's 2017 postmortem on a missing `down` migration; Flyway, Liquibase, and Sqitch all require reversibility by default). A failed deploy that needs rollback on `helios-cli` requires manual SQL reconstruction. The fleet has **two patterns** (paired up/down, and single-file with `-- DOWN`) — the third "no DOWN at all" pattern is a violation of both conventions.

## Recommended action

1. **P1 immediate (this week):** `helios-cli/migrations/0001_init.sql` — add `0001_init.down.sql` with the reverse DDL (DROP TABLE in reverse FK order). This is the highest-priority item because it is a production schema with no rollback path.
2. **P1 immediate:** `pheno/crates/agileplus-sqlite/src/migrations/016_git_bindings.sql` and `pheno/agileplus/.../016_git_bindings.sql` — add a `-- DOWN` block (or extract to up/down pair) to match the sibling 16 migrations in the same dir.
3. **P2 this sprint:** `Tokn/ledger/unified_model_provider_ledger.schema.sql` and `unified_model_provider_pareto.view.sql` — add `-- DOWN` blocks. Seeds (`*.seed.sql`) are exempt but should be annotated with `-- NO-DOWN: seed data is forward-only by design`.
4. **Policy gate (ADR-040-style):** add a migration lint that flags any `*.sql` in a `migrations/` dir that lacks either a sibling `*.down.sql` or a `-- DOWN` block. Exempt seeds via path-based rule (`*seed*.sql`).
5. **Document:** add `docs/migration-policy.md` codifying the dual pattern (up/down pair preferred, single-file with `-- DOWN` block acceptable, no-DOWN forbidden except seeds).
