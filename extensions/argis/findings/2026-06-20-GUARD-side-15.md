# Guardrail: fuzz tests for rule engine edge cases (side-15)

**Date:** 2026-06-20
**Owner:** pheno-policy-engine circle
**Status:** Proposed (FLEET_DAG.db: side-15 DONE — file artifact pending)

## Current state

The rule engine (`pheno-policy` / `phenotype-rule-engine`) ships ~40 unit
tests, all hand-written against curated example policies. Edge cases — empty
rule sets, 1000-deep ASTs, mismatched predicate arities, unicode in
identifiers — slip through because no human writes those tests. One such
bug shipped in 2026-Q1 as a post-mortem panic on malformed input.

## Proposed guardrail

A `cargo-fuzz` target covering `eval(policy, fact_set)` over arbitrary byte
inputs deserialized via `arbitrary`. Runs nightly on the `phenotype-policy`
repo with a 5-minute time budget per target; crashes get auto-filed.

## Concrete implementation

`fuzz/fuzz_targets/eval.rs` (8 LOC):

```rust
#![no_main]
use libfuzzer_sys::fuzz_target;
use pheno_policy::{Policy, FactSet, eval};

fuzz_target!(|data: &[u8]| {
    if let (Ok(p), Ok(f)) = (
        serde_json::from_slice::<Policy>(data),
        serde_json::from_slice::<FactSet>(&data[data.len()/2..]),
    ) { let _ = eval(&p, &f); }
});
```

`.github/workflows/fuzz.yml` (10 LOC):

```yaml
name: fuzz
on: { schedule: [{ cron: "0 6 * * *" }] }
jobs:
  fuzz:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: cargo install cargo-fuzz
      - run: cargo +nightly fuzz run eval -- -max_total_time=300
```

## Effort

**L** — depends on `serde(arbitrary)` derives on `Policy`/`FactSet`, one
night of CI time, a triage playbook for the first 5 corpus findings.
