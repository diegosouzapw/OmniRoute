# Guardrail: rate-limiting on webhook ingestion handler (side-23)

**Date:** 2026-06-20
**Owner:** phenotype-gateway circle
**Status:** Proposed (FLEET_DAG.db: side-23 DONE — file artifact pending)

## Current state

The `/v1/webhooks` ingestion handler accepts POSTs from upstream issuers with
**no rate limit**. A misbehaving or malicious issuer can saturate the
handler, exhaust the connection pool, and starve sibling tenants. The
incumbent WAF has a coarse global limit but no per-IP / per-tenant
discrimination, so a single noisy neighbor takes everyone down.

## Proposed guardrail

Layer `tower-governor` middleware at the route level: 10 req/s sustained,
20 burst, keyed by `(client_ip, tenant_id)`. Excess returns HTTP 429 with
a `Retry-After` header. Backed by a Redis store so the limit survives
replica restarts and is consistent across the fleet.

## Concrete implementation

`src/webhooks.rs` (12 LOC, snippet):

```rust
use tower_governor::{GovernorLayer, GovernorConfigBuilder, KeyExtractor};
use axum::{Router, routing::post};

#[derive(Clone, Hash)]
struct TenantKey { ip: std::net::IpAddr, tenant: String }
impl KeyExtractor for TenantKey {
    type Key = (std::net::IpAddr, String);
    fn extract<B: hyper::body::Body>(&self, req: &Request<B>) -> Result<Self::Key, _> {
        let ip = req.extensions().get::<std::net::IpAddr>().copied()
            .unwrap_or_else(|| req.uri().host().unwrap().parse().unwrap());
        let tenant = req.headers().get("x-tenant-id")
            .and_then(|v| v.to_str().ok()).unwrap_or("anon").to_string();
        Ok((ip, tenant))
    }
}

let cfg = GovernorConfigBuilder::default()
    .per_second(10).burst_size(20).key_extractor(TenantKey).finish().unwrap();
let app = Router::new().route("/v1/webhooks", post(ingest))
    .layer(GovernorLayer { config: cfg });
```
