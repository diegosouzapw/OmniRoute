# Guardrail: deny.toml audit for unmaintained crates (side-06)

**Date:** 2026-06-20
**Owner:** pheno-ci-templates circle
**Status:** Proposed (FLEET_DAG.db: side-06 DONE — file artifact pending)

## Current state

Repositories depend on crates from crates.io without an automated gate that
flags unmaintained or archived upstream packages. A maintainer walking away
silently shifts the maintenance burden onto us; an archived crate becomes
dead-weight that survives only because nothing notices. Today's only check
is `cargo audit` (ADR-042 monthly cadence) which catches CVEs but not
abandonment.

## Proposed guardrail

Add `deny.toml` at the workspace root with a `[maintenance]` block that fails
the CI gate when a transitive dep is marked `unmaintained` or `archived` in
the RustSec advisory DB / crates.io metadata. Wired into `pheno-ci-templates`
so every pheno-* repo inherits it on next refresh.

## Concrete implementation

`deny.toml` (workspace root, ~12 LOC):

```toml
[advisories]
db = "https://github.com/rustsec/advisory-db"
yanked = "warn"
unmaintained = "warn"

[licenses]
allow = ["MIT", "Apache-2.0", "BSD-3-Clause", "ISC", "Zlib"]

[bans]
multiple-versions = "warn"
wildcards = "deny"

[maintenance]
status = "error"          # fail CI on unmaintained / archived
age = "1 year, 6 months"  # warn if no release within window
```

`.github/workflows/ci.yml` step (5 LOC):

```yaml
- name: cargo-deny
  run: |
    cargo install --locked cargo-deny
    cargo deny check
```

## Effort

**S** — drop in `deny.toml`, install `cargo-deny` once in the runner image,
copy the workflow step. Trivial to roll out across all pheno-* repos via the
shared CI template.
