# Guardrail: pre-commit hook for cargo fmt / clippy (side-50)

**Date:** 2026-06-20
**Owner:** pheno-ci-templates circle
**Status:** Proposed (FLEET_DAG.db: side-50 DONE — file artifact pending)

## Current state

`cargo fmt --check` and `cargo clippy -D warnings` only run in CI. A
contributor pushing 3 commits with bad formatting waits 6 minutes for the
runner to tell them what they could have seen locally in 4 seconds. The
fast feedback loop lives in the developer's terminal, not the CI queue.

## Proposed guardrail

Add a `pre-commit` hook via the existing `lefthook.yml` that runs
`cargo fmt --all -- --check` and `cargo clippy --workspace --all-targets
-- -D warnings` on staged files only. Hook is parallel-safe (cargo's own
target dir handles the lock). Bypassable via `LEFTHOOK=0 git commit ...` for
emergencies.

## Concrete implementation

`lefthook.yml` pre-commit block (10 LOC):

```yaml
pre-commit:
  parallel: true
  commands:
    fmt:
      glob: "*.rs"
      run: cargo fmt --all -- --check
    clippy:
      glob: "*.rs"
      run: cargo clippy --workspace --all-targets -- -D warnings
    deny:
      glob: "*.toml"
      run: cargo deny check
```

Install once per clone: `lefthook install` (the existing repo Makefile
target already wires this).

## Effort

**S** — 15 minutes per repo. Test by staging a deliberate format violation
and confirming the hook rejects before commit.
