# side-41: Missing `.dockerignore` files

**Date:** 2026-06-20
**Scope:** All directories containing a `Dockerfile*` (excludes `.git/`, `target/`, `node_modules/`, `.venv/`).

## What I checked

```bash
# 1. Total Dockerfiles in the meta-repo
find . -name "Dockerfile*" -not -path "*/.git/*" -not -path "*/target/*" \
     -not -path "*/node_modules/*" | wc -l
# → 213

# 2. Existing .dockerignore files
find . -name ".dockerignore" -not -path "*/.git/*" -not -path "*/target/*" \
     -not -path "*/node_modules/*" | wc -l
# → 30

# 3. .dockerignore per core substrate repo
find ./{PhenoCompose,AuthKit,Configra,phenotype-hub,phenotype-bus,phenotype-gateway,phenotype-registry,phenotype-ops,phenotype-tooling,helios-cli,heliosBench} -name ".dockerignore" 2>/dev/null | wc -l
# → 1 (only PhenoCompose/.dockerignore)
```

## Findings

**Has `.dockerignore` (30 dirs):**
`BytePort/.dockerignore`, `PhenoProject/rust/Planify/.dockerignore`, `PhenoProject/rust/Planify/apps/{web,admin}/.dockerignore`, `nanovms/.dockerignore`, `hwLedger/.dockerignore`, `FocalPoint/AtomsBot-2nd/.dockerignore`, `FocalPoint/.dockerignore`, `OmniRoute-cluster-profiles/.dockerignore`, `thegent/.dockerignore`, `HexaKit/.dockerignore`, `./.dockerignore` (monorepo root), `AgilePlus/AgilePlus-wtrees/orch-v12-s2-001/.dockerignore`, `AgilePlus/AgilePlus-wtrees/orch-v12-s4-018-cargo-deny/.dockerignore`, `AgilePlus/.dockerignore`, `Planify/.dockerignore`, `Planify/apps/{web,admin}/.dockerignore`, `OmniRoute-combos-split/.dockerignore`, `PlayCua/.dockerignore`, `cliproxyapi-plusplus/.dockerignore`, `cliproxyapi-plusplus/.worktrees/tier-0-hygiene/.dockerignore`, `QuadSGM/.dockerignore`, `Tracera/{frontend/apps/web,backend,.devcontainer}/.dockerignore`, `Tracera/.dockerignore`, `PhenoCompose/.dockerignore`, `OmniRoute/.dockerignore`.

**Missing `.dockerignore` for active Dockerfiles (P1):**
- `helios-cli/Dockerfile` (no .dockerignore) — full `helios-cli/` will be COPYed including `target/`, `node_modules/`, `.git/`.
- `phenotype-ops/review-surface/Dockerfile` (no .dockerignore) — `phenotype-ops/` parent will be sent, including `python/`, `docs/`, archive dirs.
- `phenotype-ops/agent-devops-setups/llama-cpp/Dockerfile` (no .dockerignore) — model files (.gguf, often 4-40 GB) will leak in if path is in build context.
- `phenotype-tooling/crates/policystack/Dockerfile` (no .dockerignore) — entire `phenotype-tooling/` workspace (47+ crates, ~50k files) will COPYed by default.
- `phenotype-tooling/docs/absorbed-from-kwality/src/engines/runtime-validator/Dockerfile` (no .dockerignore).
- `phenotype-tooling/docs/absorbed-from-dispatch-mcp/docker/Dockerfile.llama` (no .dockerignore).
- `phenotype-tooling/docs/absorbed-from-kodevibe/engine/Dockerfile` (no .dockerignore).
- `Tokn/Dockerfile`, `slm-server/Dockerfile` (no .dockerignore).
- `pheno/Dockerfile.rust`, `pheno/python/Dockerfile.python`, `pheno/agileplus/Dockerfile.rust`, `pheno/agileplus/python/Dockerfile.python`, `pheno/agileplus-mcp/Dockerfile`, `pheno/agileplus/agileplus-mcp/Dockerfile`, `pheno/tests/integration/Dockerfile.test` (no .dockerignore).
- `HexaKit/.../Dockerfile*` (5 files, no .dockerignore — though `HexaKit/.dockerignore` exists at root and partially covers).
- `PhenoDevOps/.../Dockerfile*` (6 files, no per-dir .dockerignore).
- `AgilePlus/.../Dockerfile*` (template files; `AgilePlus/.dockerignore` exists at root).
- `Dino/Dockerfile`, `DinoForge-UnityDoorstop/.../Dockerfile` (no .dockerignore).
- `argis-extensions/.../Dockerfile`, `argis-extensions/services/{promptadapter,researchintel}/Dockerfile` (no .dockerignore).
- `Authvault/Dockerfile`, `cliproxyapi-plusplus/.worktrees/.../Dockerfile` (cliproxyapi-plusplus/.dockerignore covers root; worktree copy may diverge).
- `Tracera/ARCHIVE/DOCKER/Dockerfile`, `Tracera/ARCHIVE/CONFIG/.../Dockerfile` (archive content; out of active scope).

Approximately **170+ of 213 Dockerfiles (80%)** lack a co-located `.dockerignore`. For builds that `COPY . .` this means `target/`, `node_modules/`, `.git/`, `*.log`, build artifacts, secrets, and model weights are all sent to the daemon. Result: large build contexts (often 1-10 GB), slow `docker build` times, and inadvertent secret leakage (e.g., `.env` files in COPY).

## Severity

**P1.** Missing `.dockerignore` is on the OWASP Docker Top 10 and CIS Docker Benchmark (4.7). It is rarely a P0 because the secrets-not-leaked pattern is a separate concern, but for `phenotype-ops/agent-devops-setups/llama-cpp/Dockerfile` and `phenotype-tooling/crates/policystack/Dockerfile` (polycrate workspaces with 10k+ files) the build context alone is a 5+ GB leak per build invocation. Not P0 because no active leak has been observed; this is a hygiene gap.

## Recommended action

1. **Wave 1 (this sprint):** Add `.dockerignore` to the 8 active service repos that are missing it (`helios-cli`, `phenotype-ops/review-surface`, `phenotype-ops/agent-devops-setups/llama-cpp`, `phenotype-tooling/crates/policystack`, `Tokn`, `slm-server`, `argis-extensions/*`).
2. **Minimum-content policy:** every `.dockerignore` must include: `.git`, `target`, `node_modules`, `.venv`, `__pycache__`, `*.log`, `.env`, `.env.*`, `*.gguf`, `*.bin`, `dist`, `build`, `coverage`, `.cargo`, `.pytest_cache`, `.ruff_cache`, `.mypy_cache`.
3. **Template:** provide a `thegent/templates/operational/docker/.dockerignore.template` and reference it from each Dockerfile header.
4. **Policy gate (ADR-040-style):** `policy-gate.yml` step that greps for `.dockerignore` in every directory with a `Dockerfile*` and fails if missing.
5. **Tracera/ARCHIVE excluded:** archived Dockerfiles are out of scope; only active build paths need the gate.
