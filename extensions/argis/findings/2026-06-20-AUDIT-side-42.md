# side-42: CORS restrictions on public endpoints

**Date:** 2026-06-20
**Scope:** CORS configuration in active API services (Rust/Go/Python/TypeScript), excluding `.venv/` and `node_modules/`.

## What I checked

```bash
# 1. Find CORS config in Go services
grep -nE "AllowedOrigins|AllowAllOrigins|origins" \
     ./BytePort/backend/server.go ./KWatch/config/config.go \
     ./KWatch/server/server.go ./KWatch/server/middleware.go
# Findings (see below)

# 2. Find CORS config in Python (FastAPI)
grep -n "allow_origins" ./Parpoura/venture/api/main.py
# Line 37: allow_origins=["*"]

# 3. Find CORS in TypeScript / frontend proxies
grep -rE "Access-Control-Allow-Origin|cors|allowedOrigins" \
     . --include="*.ts" --include="*.js" --include="*.go" \
     | grep -v "node_modules|/.venv/|/.git/" | head -20
```

## Findings

**P0 violation — wildcard CORS in production API:**
- `Parpoura/venture/api/main.py:37` — `allow_origins=["*"]` is a literal `*` for the FastAPI CORS middleware. Any browser on any origin can call this API. If the API has any cookie-based auth or accepts credentials, this is an OWASP A05:2021 (Security Misconfiguration) **critical** finding. If the API is unauthenticated, it remains a P1 hardening issue.

**P0/P1 violation — wildcard CORS as default config:**
- `KWatch/config/config.go:275` — `DefaultServerAllowedOrigins  = "*"`. The default for the CORS allowed-origins config is the literal wildcard. Even if operators override this in deployment, the **default ships as `*`**, which is the wrong default for a security-relevant config.
- `KWatch/config/config.go:353` — `AllowedOrigins:  []string{DefaultServerAllowedOrigins}` (the default value flows into the struct init).
- `KWatch/config/config.go:90-93` — `AllowedOrigins []string` with comment "specifies allowed origins for CORS requests" — no minimum-security mention.
- `KWatch/server/server.go:54` — `log.Printf("CORS enabled for origins: %v", s.config.AllowedOrigins)` — emits the list at startup; no enforcement that `*` is rejected.

**Acceptable — explicit allowlist:**
- `BytePort/backend/server.go:27,89-99` — `parseAllowedOrigins()` parses from config and defaults to an explicit allowlist (not `*`). The comment block at line 99+ shows the default. This is a safe pattern.
- `AuthKit/go/middleware/middleware.go:415+` — uses a `CORSConfig` struct with explicit `AllowedOrigins []string`; no `*` default found. Good.

**Not yet checked (lower priority):**
- `helios-cli/harness/src/harness/config.py` — has CORS config; not yet fully inspected. Recommend v12 audit.

## Severity

**P0** for `Parpoura/venture/api/main.py:37` (wildcard CORS in production API). **P1** for `KWatch/config/config.go:275` (default-to-wildcard).

A wildcard CORS policy is OWASP A05:2021 (Security Misconfiguration) and is the single most common CORS-related CVE vector. If `Parpoura/venture/api/main.py:37` is reachable on the public internet, it is exploitable today for any browser-based attack (CSRF via `*` + `credentials: include` is a special case, but `*` alone is a P0 because it makes the API indistinguishable from a public endpoint by any cross-origin attacker).

## Recommended action

1. **P0 immediate (today):** `Parpoura/venture/api/main.py:37` — replace `allow_origins=["*"]` with `allow_origins=settings.ALLOWED_ORIGINS` (read from env var, e.g. `https://parpoura.app,https://admin.parpoura.app`). Default to empty list, not `*`. If credentials are not used, the change is one-line; if credentials are used, also set `allow_credentials=True` against the explicit list and `allow_headers=["*"]` is OK.
2. **P0 immediate:** `KWatch/config/config.go:275` — change `DefaultServerAllowedOrigins  = "*"` to `DefaultServerAllowedOrigins  = ""` and add a startup check that fails if `len(AllowedOrigins) == 0` in production mode (allow empty in dev).
3. **P1 (this week):** add a CORS policy gate to `policy-gate.yml` that greps for `allow_origins=\["\*"\]` / `AllowedOrigins.*=.*"\*"` and fails the PR.
4. **P2 (this sprint):** add `helios-cli/harness/src/harness/config.py` to the audit list and verify the same pattern.
5. **Document the policy:** ADR-style doc `docs/cors-policy.md` defining "default deny, explicit allowlist, no wildcards in production".
